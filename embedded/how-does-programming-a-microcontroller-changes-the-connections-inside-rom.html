How does programming a microcontroller changes the connections inside ROM

up vote 3 down vote favorite

I know what programming is... It is to erase the existing program on ROM and then flash the required program into ROM.

I know about flash ROM, which programs the flash type ROM block by block. We send electrical signals and boom! your program is dumped into the microcontroller. Can anyone explain me how this happens?

I also know that ROMs are programmable devices and the fuses are burnt and made again.... but how is it done (please answer this specifically electrically rather than UV exposure method as in EPROM)?

Thankyou for spending your valuable time.

Most embedded systems today use flash memory for the code, same as is used in SD cards / SSDs etc. It is cheaper (per byte) than traditional EEPROM and can be rewritten many times. The flash memory is accessed via a flash controller that does the actual voltage-fiddling necessary to erase or write data into the flash memory. For technical reasons erases must be done a page at a time with flash, and there are some restrictions on writes as well, but from the programmer's perspective it is the same as using any other API that talks to the hardware.

Some chips have special write-once memory, useful for security, serial numbers etc.

At the physics level, flash memory uses quantum tunneling. When the control gate voltage is high enough, electrons can be moved in/out of the floating gate. At lower voltages, they are stuck there.

Difference between USART_TXC interrupt and USART_UDRE interrupt

up vote 5 down vote favorite

What is the difference between the USART_TXC (Transmit Complete) and USART_UDRE (UART Data Register Empty) interrupts in AVR?

When/Why should I use one over the other?

Inside the USART, there is circuitry to hold one piece of character-based data (typically 8-bits these days) from the application, and a shift register that sends it out onto the transmission line one bit at a time (and some bit clock generation and error detection and so on).

The UDRE interrupt occurs when the USART copies the data to be transmitted into the transmit shift-register. This either happens after the USART has finished transmitting the previous piece of data (if any), so it indicates that the USART is ready to accept more data for transmission.

The TXC interrupt occurs when the USART has finished transmitting the last stop bit of the onto the data line. If your USART is configured for 1 start bit, 8 data bits, no parity and 2 stop bits, it will be at least 11 bit intervals after the UDRE interrupt.

You monitor UDRE so that you don't load data into the UDRn register before the previous piece of data has been sent, which would cause you to skip data.

You monitor TXC so that you know when you've finished sending data. That allows you to turn off the USART to save power, for example, or to switch from transmit to receive modes in a half-duplex scenario.

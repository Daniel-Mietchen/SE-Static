How can I split a memory section into different physical memory zones?

up vote 10 down vote favorite

I'm using a Cortex R5F processor which features two Tightly Coupled Memory (TCM) blocks, each 4KB in size. My code generates 7KB of read-write (RW) data. I want to put this RW data into TCM memory but a single TCM can't accommodate this size.

Is there any way to split the RW section between the two TCM blocks, which do not have contiguous addresses?

I'm compiling with armcc from DS5.

We've had to do something like this before. We used a scatter file to tell the linker to put RW/ZI stuff in specific locations. Granted, we were using uVision (don't judge, please). Anyways, uVision uses armcc, armasm, and armlink, so the following should work. And if it doesn't, I guess there's a significant difference between uVision and DS5 (or between Cortex-M3 and Cortex R5F).

Here's a sample scatter file:

#! armcc -E -I. --cpu Cortex-M3
#include "Map.h" ; Use this if you define memory locations in a specific file

LR_PRIMARY_APPLICATION START_ADDRESS ROM_SIZE {
  ER_APPLICATION +0 {
    *.0 (RESET, +First)
    *(InRoot$$Sections)
    .ANY (+RO)
  }

 GENERAL_RAM RAM_START GENERAL_RAM_SIZE {
   *(GeneralRam)
   .ANY (+RW +ZI)
 }

 TCM_1 TCM1_START TCM_SIZE {
   *(SectionTCM1)
 }

 TCM_2 TCM2_START TCM_SIZE {
   *(SectionTCM2)
 }

}

You'll need to define addresses and sections (TCM1,TCM2) in the "Map.h" file. Note that the linker doesn't like fancy address/length definitions, so you're definitions should just be the hex number (obviously the specific numbers are made up):

/* Section attribute defintions */
#define TCM1_RW __attribute((section("SectionTCM1"),used))
#define TCM2_RW __attribute((section("SectionTCM2"),used))

/* Address definitions */
#define TCM1_START 0x20001000
#define TCM2_START 0x20002000
#define TCM_SIZE   0x00001000

And you'll want to put variables in specific TCM blocks like so:

uint8_t   SomeVar[256] TCM1_RW;
uint32_t  OtherVar[16] TCM2_RW;

ARM documentation is a little unclear here, but is still pretty good: http://www.keil.com/support/man/docs/armlink/armlink_pge1362065968963.htm

Hope all this helps.

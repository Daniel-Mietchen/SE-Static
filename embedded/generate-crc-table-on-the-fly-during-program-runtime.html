Generate CRC table on the fly during program runtime

up vote 3 down vote favorite

I am trying to reverse engineer a communication protocol between to PIC micro-controller boards. One of the challenge is we have some documentation stating that it uses 8 bit CRC. But no information about polynomial is mentioned.

We want to try brute-force method of taking packet and try all 16 bit algorithms and check if CRC transmitted by board matches calculated. So that we can derive CRC polynomial.

For speed we want to use table based CRC calculations. But every test polynomial we need to generated CRC table. Is there any that we can generate this 256 bytes on the fly by some function?

It is possible to generate the table on the fly. The following is an adaptation of a CRC-32 table generator I used recently.

uint8_t lookupTable[256]; /* This should match the width of the POLY. */
uint8_t crcOutput; /* Placeholder for crc generation. */
uint32_t b; /* This needs to be wider than 8 bits. */
uint8_t count; /* Iterator */

/* Initialize byte counter, b. */
b = 0;

/* Generate each table entry, based on byte counter, b. */
while (b < 256) 
{
    crcOutput = b;
    for(count = 0; count < 8; ++count)
    {
        crcOutput = (crcOutput >> 1) ^ (((crcOutput & 0x01) > 0) ? POLY : 0x00);
    }
    lookupTable[b] = crcOutput;
    ++b;
}

Python's crcmod library can generate objects which compute the CRC for a given width and polynomial, and it can also generate C code to perform the computation.

This gives you a choice of either capturing some packets and analysing them with python on the desktop, or of using python to generate some C code to run on the PIC.

One thing to watch out for: Occasionally a protocol might checksum not an entire packet but just the data 'payload' within. So if you don't know otherwise, and if an exhaustive search doesn't find a working polynomial, you may need to start working out which bytes might not be included in the checksum.

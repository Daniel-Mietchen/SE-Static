How to boot OLinuXino from a USB stick?

up vote 3 down vote favorite

I have the A13-OLinuXino board.

It booted from the SD card successfully, but the boot process does not start if I try to boot from a USB stick.

How do I boot OLinuXino from a USB stick?

The only approach I can find to make this work is to take advantage of special operating mode of the Allwinner SOC called FEL mode. (I can't find what the acronym actually stands for.)

FEL Mode is "a low-level subroutine contained in the BootROM on allwinner devices. It is used for initial programming and recovery of devices using USB." By switching into this debug mode, you can get the SOC to boot via the USB OTG port. It does take some special configuration of the OS and bootloader to make it work though, and I can only find information about preparing an image of Sunxi Linux for it.

This is the official guide: http://linux-sunxi.org/FEL/USBBoot. I think the process is too long to repeat here, so you'll have to consult this guide.

If anything is missing, here is another tutorial I found on this topic: http://docs.cubieboard.org/tutorials/cb2/development/booting_from_usb_in_fel_mode

DISCLAIMER: I have never booted an A13-Olinuxino.

The answer to this question depends on your current setup, but for now I am going to assume that you are using uboot as your current bootloader. There may be some slight differences from the version of uboot that I am familiar with and the version that the A13-Olinuxino uses, so be warned.

If you check the files on the boot partition of your SD card, you should find a file named uEnv.txt. This file is the uboot bootscript and it contains the boot configuration. More information on this file can be found here. This file will be the key to booting from USB, so be sure to make a backup somewhere. Also, please note that I am assuming your USB stick has been properly formatted.

To get started, make sure that your SD card is inserted, and you are connected to the serial debug port on the board. For the A13-Olinuxino, the user's manual suggests that the USB Mini B connector can be used for this.

  1. Begin by plugging your A13-Olinuxino into a separate computer using the USB Mini B connector. (Note: If you do not see the debug output from step 2, then try using the UART1 connector with a USB-to-UART).

  2. You are going to need a serial terminal, if you are using windows I recommend TerraTerm, and for linux Minicom should suffice. Connect your USB-to-UART, and open the correct port with your terminal software of choice (you can jumper the TX and RX pins to test which port is connected). With your A13-Olinuxino connected, apply power to the board (again, make sure your SD card is inserted because we are going to be booting from it initially). If everything is connected properly, you should see the uboot prompt in your terminal.

  3. The following is the output from the version of uboot that I use (an AM335X board), your output should be similar:

U-Boot 2013.10-g6c6a700 (Nov 20 2013 - 16:56:02)

AM335X-GP rev 0
I2C:   ready
DRAM:  256 MiB
WARNING: Caches not enabled
NAND:  Using 2k bch8 layout
ECC Mode = 2 lo = 56/0
256 MiB
MMC:   OMAP SD/MMC: 0, OMAP SD/MMC: 1
Net:   Could not get PHY for cpsw: addr 1
Could not get PHY for cpsw: addr 1
cpsw
Hit any key to stop autoboot:  0
U-Boot#

At Hit any key to stop autoboot: press a key. You should get a U-Boot# prompt. From here you can run the command printenv to display the contents of the uEnv.txt file (actually is prints the current environment, but since that was loaded from the uEnv.txt file...).

  1. Within the large wall of text from printenv (and it will be a wall), there should be a variable bootcmd. This is the piece of information that uboot uses to decide which peripheral to boot from whether it be NAND flash, MMC, USB, Ethernet, I2C, SPI etc. Since I have never seen the uboot configuration from the A13-Olinuxino, I can only give you a rough idea of what needs to be changed. The bootcmd variable for my current board is as follows:

bootcmd=if mmc rescan; then echo SD/MMC found on ${mmc_dev};run mmc_boot;fi;run nand_boot;

This tells uboot to search for an MMC, to boot from it if one is found, and if not boot from NAND flash. In this case mmc_boot and nand_boot are other variables present in my uEnv.txt file, so please do not just copy this to your file.

  1. This is where you will need to modify the bootcmd value. This page should provide what you will need to know, but essentially your bootcmd variable should resemble something like this: usb start;fatload usb 0 61000000 uImage;bootm. You can set the current environment variable by using either the editenv command, or askenv. The editenv command will allow you to do an in-place edit of the command, and askenv will ask you for the complete command.

For example:

askenv bootcmd
usb start;fatload usb 0 61000000 uImage;bootm
printenv bootcmd

Will allow you to set, and then verify the contents of the bootcmd variable. Please write down, copy, or memorize the changes you make here, they are not permanent!

  1. Once you have set the variable in memory using either the setenv or editenv commands, insert the USB stick and execute run bootcmd. This will use the uboot command parser to run the contents of the bootcmd variable. If this process is successful, you should see linux boot. Honestly, though, I doubt that the bootcmd I have provided will work without some input from you in the comments.

  2. Now that, hopefully, you have successfully booted from USB we will be saving the environment so that the SD card is no longer needed to boot from. Power off the board and insert the SD card into your computer, we will be modifying the uEnv.txt file. From step 5, you should have a good copy of the bootcmd that worked for you, delete the current bootcmd entry, and replace it with yours. Make sure it says bootcmd= before your command.

  3. Insert the SD card into the A13-Olinuxino one last time, and boot into uboot (press a key to stop autoboot), and run the printenv command. Verify that the bootcmd variable is correct, and then type saveenv. The saveenv command should save the current environment to the NAND flash chip present on the A13-Olinuxino. Finally, power off the board, remove the SD card, and insert the USB stick. If all has gone well, the following boot process should load the environment from NAND, run your bootcmd, and boot linux from your USB drive.

A few things: It is generally a bad idea to enforce one single boot option. If you remember the bootcmd from my board, you will see that I boot from the SD card if it is present. Looking over the schematic for the A13-Olinuxino, there does not appear to be any boot mode jumpers. I cannot speak to the A13 SoC, but there are some devices that require a boot mode configuration (such as the AM335X), so if you get stuck that could by why.

By the way, if you are looking for a getting started tutorial, this one looks great.

Killing a process group under busybox

up vote 3 down vote favorite

I use start-stop-daemon to start a service that writes various temp files to disk. The daemon makes use of other executables (e.g. zip, find) and cleans up by backgrounding (using &) a cleanup task. Unfortunately stopping the daemon with start-stop-daemon doesn't stop child processes, with the result that the partition they are touching can't be unmounted.

Here is an example of the daemon:

$ ps -o pgid,pid,ppid,args
PGID  PID   PPID  COMMAND
    1     1     0 init
 3687  3688     1 {mydaemon} /opt/mydaemon
 3687  3712     1 {exe} ash /opt/deletestuff /temp
 3687  3719  3712 /usr/bin/find /temp -mindepth 1 -type d -depth -exec rmdir {} ;

Ideally there would be an option to pass to start-stop-daemon that killed the entire process group, but looking at the source code the only way I see do to this is to run as a unique user and kill using the userspec option.

Under "regular" linux one would kill the process group using kill with a negated process group ID. Busybox kill doesn't support this. Busybox pkill -P will kill based on PPID but that won't work either since deletestuff's PPID is 1. This leaves me with scraping pids out of ps or /proc and killing them off individually.

How do I kill mydaemon and all its children at one fell swoop?

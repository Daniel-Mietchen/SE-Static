Is it possible to disable the Photon's status LED?

up vote 2 down vote favorite

I bought a Photon board from Particle. Everything is working fine, but the status LED is always on or blinking. Even when running my own application, the LED is slowly 'breathing'. This is annoying, because it is needless power consumption and may confuse end-users. Is there any way to disable the status LED from within an application?

To me, from the documentation, it seems like: yes, it should be possible.

From the schematic in section 6.3 the Blue status LED is connected to D7 and an example of theirs connects to that pin to turn the LED on and off, so it would be weird if it then kept blinking anyway. It's possible they turn it on by default until you take control, of course.

I do not own the module, so I cannot say for 100% certain, but the documentation on their website seems to imply that you can just take control of pin D7 and turn the LED off by making that pin low.

There is a small chance that their FreeRTOS modification also uses the LED on D7 for some status-output, but I can only imagine that being an oversight if it cannot be disabled, which would probably be a good reason for creating a support-ticket. If that were intentional and not intended to be disabled when you want to use it, it should certainly be mentioned in the examples that use the LED themselves.

EDIT:

I misread the website vis-a-vis the RGB LED and didn't know the videos I saw were of status manipulation, I thought I saw a vid where a button dimmed the RGB LED.

Unfortunately it does seem now that they do not expose any functions for controlling it. I cannot find any Git links for adapted FreeRTOS types, so it seems they're holding all that a bit "closer to the vest" as we say here. But that may be just my oversight. Of course adding functionality for that is easiest for a principle designer/engineer upon request, but if FreeRTOS sources are available, it should be "easy enough" to expose new functions to be able to control the three GPIO pins on the WiFi module directly and so block the LED.

However, at first glance now, the quickest, if you are handy in that sense, would be to desolder R1, R2 and R7 that drive the LED. But that is, of course not a production-type thing to do. Then again, if you contact them about ordering 100's or 1000's of pieces for a full run, on the one condition that you will be able to turn that darn LED off, that might be some motivation for support in that sense, that buying just 2 would be to a much, much lesser extend.
